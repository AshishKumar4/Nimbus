## 1. Root-cause analysis

### Failure 1 ‚Äî `Cannot read module: home/user/app/node_modules/ret/dist/types` (fastify)

**Stack-of-evidence:**

1. Error string emitted at `src/node-shims.ts:2024`:
   ```js
   if (code === null) throw new Error("Cannot read module: " + resolvedPath);
   ```
2. `__loadModule` reaches that line because `__readFileOr(resolvedPath, null)` returned `null` ‚Äî the resolved path is **not a file**.
3. `__loadModule(resolvedPath)` is called with the result of `__resolveFile(base)` (`src/node-shims.ts:1844`).
4. `__resolveFile` iterates `exts = ["", ".js", ".mjs", ".cjs", ".json", "/index.js", ...]` and returns the first hit via `__fileExists`:
   ```js
   for (const ext of exts) { if (__fileExists(base + ext)) return base + ext; }
   ```
   The first ext is `""` (empty string).
5. `__fileExists` (`src/node-shims.ts:1832`) returns `true` for a directory in three ways:
   - `k in __vfsDirs` ‚Äî line 1836
   - prefix scan over `__vfsBundle` ‚Äî lines 1838‚Äì1841 (any nested file makes the dir "exist")
   - `k in __vfsWrites` ‚Äî line 1835 (runtime-created dirs)

   So when `base = node_modules/ret/dist/types` (a directory containing `index.js`), `__fileExists(base + "")` returns true via the prefix scan, and `__resolveFile` returns the bare directory path. The `/index.js` ext is **never tried**.
6. `__loadModule` then calls `__readFileOr(directory, null)` ‚Üí null ‚Üí throws.

**Why prefetch (`require-resolver.ts:62`) does NOT have the bug:** that loop guards with `&& !vfs.isDirectory(p)` (line 66). So the bundle correctly contains `ret/dist/types/index.js` content. But the runtime resolver returns the wrong key.

**Sibling failures** (audit/probes/packages-prod-w26a/*):
- `redis` ‚Üí `@redis/client/dist/lib/client` directory require.
- Any package that does `require('./lib/foo')` where `foo` is a directory.

**Verdict:** real bug. Two-line ordering / strict-file-check fix.

### Failure 2 ‚Äî `tldts/dist/es6/index.js: file was not pre-bundled` (jsdom)

**Stack-of-evidence:**

1. Error string emitted at `src/node-shims.ts:2058`:
   ```js
   if (evalErr.message && evalErr.message.includes("Code generation from strings disallowed")) {
     throw new Error("Cannot load module '" + resolvedPath + "': file was not pre-bundled. Add it to the VFS bundle.");
   }
   ```
2. That error fires only when `__compiledModules.get(resolvedPath)` returned `undefined` AND the runtime fallback `new Function(code)` threw the workerd CSP error.
3. `__compiledModules` is populated at facet-startup time:
   `src/facet-manager.ts:205‚Äì210` and `383‚Äì388`:
   ```js
   for (const [__p, __c] of Object.entries(__MODULE_VFS_BUNDLE)) {
     if (__p.endsWith(".js") || __p.endsWith(".mjs") || __p.endsWith(".cjs")) {
       try { __compiledModules.set(__p, new Function("exports","require","module","__filename","__dirname", __c)); } catch {}
     }
   }
   ```
4. `new Function(body)` parses the body as a **function** body, not a Module. Top-level `import`/`export` STATEMENTS throw `SyntaxError`. The `try { ‚Ä¶ } catch {}` swallows the error silently ‚Äî no diagnostic is emitted, no warning, no counter bump. The map entry is simply absent.
5. At request time, `__loadModule` falls through to a second `new Function` (line 2054) which workerd's CSP blocks (`Code generation from strings disallowed`), and the catch at 2057 surfaces "file was not pre-bundled".

**Why the file lands in the bundle in the first place:** any of three routes:
- (a) explicit subpath require: `require('tldts/dist/es6/index.js')` somewhere in the require chain;
- (b) `greedyAddMainEntries` (`src/facet-manager.ts:576`) iterates `meta.module` and `dot.import` candidates (lines 614, 621) ‚Äî for ESM-only packages those are the only candidates.
- (c) jsdom directly imports the `module` field of a transitive dep that has both ESM and CJS but where the dep's own `package.json#exports` is structured oddly.

**Why does tldts hit this?** tldts is dual-package:
```
"main":   "./dist/cjs/index.js",
"module": "./dist/es6/index.js",
"exports": {".": {"import": "./dist/es6/...", "require": "./dist/cjs/...", "default": "./dist/cjs/..."}}
```
Runtime resolver (`__NIMBUS_CJS_CONDITIONS = ['require','node','default']`) correctly picks `dist/cjs/index.js` for `require('tldts')`. But greedy oversample ALSO adds `dist/es6/index.js` to the bundle because of `meta.module`. The CJS path works fine; the ES6 path silently fails to compile and is harmless **until** something inside jsdom does `require('tldts/dist/es6/index.js')` (or the static-analysis prefetch resolves a transitive that lands on the ESM file). At that point, the silent-fail bites.

**Sibling failures** (W3 retro + audit/probes/packages-prod-w26a):
- `@remix-run/react` ‚Üí `dist/esm/index.js`
- `@tailwindcss/vite` ‚Üí `dist/index.mjs`
- `react-remove-scroll` ‚Üí `dist/es2015/index.js`
- `astro` ‚Üí `dist/index.js` (ESM-only)
- Any package whose primary or referenced export is ESM and lands in `__MODULE_VFS_BUNDLE`. The npm landscape since 2023 makes this the norm, not the exception.

**Verdict:** real bug. Three-part fix:
1. Detect ESM at facet-build time (regex sniff for top-level `import`/`export`).
2. Pre-transform ESM ‚Üí CJS at bundle-assembly time using the existing `EsbuildService.transform({ format: 'cjs' })`. Cache result keyed by hash.
3. Stop swallowing the SyntaxError at the per-file `new Function` step ‚Äî push it to a runtime-visible diagnostic for forward debugging.

---

## 2. Fix sketches (file:line targets)

### Fix A ‚Äî directory-as-index in `__resolveFile`

**File:** `src/node-shims.ts`
**Function:** `__resolveFile` (lines 1844‚Äì1850)

Two viable options:

**(A1) Skip directory hits in the empty-extension probe.**
Add a `__pathIsFile(p)` helper that returns true only for actual file content (excludes `__vfsDirs` membership and the prefix scan), then gate the empty-ext probe on it. Other call sites (`__resolveNodeModule` line 1967, `__resolveImportsField` line 1988) keep the loose `__fileExists` because they need "does this directory exist".

**(A2) Reorder the ext list so `""` is tried last.**
Cheap and surgical, but loses the "exact match wins" semantic: `require('./types.js')` would still hit `.js` as the explicit extension first because `path.endsWith(".js")` short-circuits, but for paths like `./typesjs` it might pick a `.js`-extended sibling unexpectedly.

**Decision:** **A1 (strict-file probe).** Cleanest, no semantic change to other callers, mirrors `require-resolver.ts:resolveFile`'s `&& !vfs.isDirectory(p)` guard.

```js
function __pathIsFile(path) {
  const k = path.replace(/^\/+/, "");
  if (__vfsBundle && k in __vfsBundle) return true;
  if (__vfsWrites && k in __vfsWrites) return true;
  // NB: deliberately does NOT consult __vfsDirs nor do prefix scan.
  return false;
}
function __resolveFile(base) {
  const exts = ["", ".js", ".mjs", ".cjs", ".json", "/index.js", "/index.cjs", "/index.mjs", "/index.json"];
  for (const ext of exts) {
    const cand = base + ext;
    // For the empty-ext probe (matching the path exactly), require an actual file.
    // Other ext probes already produce file-shaped paths so __fileExists is fine.
    if (ext === "") { if (__pathIsFile(cand)) return cand; continue; }
    if (__fileExists(cand)) return cand;
  }
  return null;
}
```

### Fix B ‚Äî ESM-to-CJS transform during `buildPrefetchBundle`

**File:** `src/facet-manager.ts`
**Function:** `buildPrefetchBundle` (lines 679‚Äì736)

Strategy: after the prefetch + greedy-oversample pass, walk every `.mjs` and `.js` value in `bundle`. If the source has top-level ESM markers, run `esbuildService.transform(code, { loader: 'js', format: 'cjs', target: 'esnext' })` and replace the value. Cache by content hash to avoid re-transforming on every exec.

Make `buildPrefetchBundle` **async** so we can await the transforms. Caller in `FacetManager.exec` (line 811) awaits it.

ESM detection regex (top-level `import`/`export` only, ignore comments / strings):
```js
function looksLikeEsm(src) {
  // Strip line comments and block comments cheaply (good enough for sniffing).
  const stripped = src.replace(/\/\/[^\n]*/g, '').replace(/\/\*[\s\S]*?\*\//g, '');
  // `import` STATEMENT: at line start (after ws), followed by ID/`{`/`*`/string,
  // up to a newline or `;`. Skip `import(...)` (dynamic ‚Äî always allowed in CJS).
  const importStmt = /(^|\n)\s*import\s+(['"][^'"]+['"]|[\w*$\{])/;
  // `export` STATEMENT: at line start.
  const exportStmt = /(^|\n)\s*export\s+(default\b|\{|\*|let\b|const\b|var\b|function\b|class\b|async\b|type\b)/;
  return importStmt.test(stripped) || exportStmt.test(stripped);
}
```

Wire-up:
```ts
async function buildPrefetchBundle(
  vfs, scriptPath, cwd, entryCode,
  esbuild?: EsbuildService,
): Promise<FacetVfsState> {
  // ... existing logic ...
  if (esbuild) {
    for (const path of Object.keys(bundle)) {
      const src = bundle[path];
      if (!path.endsWith('.js') && !path.endsWith('.mjs')) continue;
      if (!looksLikeEsm(src)) continue;
      try {
        const t = await esbuild.transform(src, { loader: 'js', format: 'cjs', target: 'esnext' });
        bundle[path] = t.code;
      } catch (e) {
        // Leave original ‚Äî the facet-side new Function() will fail, but at
        // least the diagnostic surface (Fix C) will tell us why.
      }
    }
    // RE-encode and re-evict if the transform pushed us over the cap.
    encoded = encoder.encode(JSON.stringify({ bundle, manifest })).length;
    if (encoded > BUNDLE_MAX_ENCODED_BYTES) { /* re-evict largest */ }
  }
  return { bundle, manifest, reachableCount: fileCount, truncated };
}
```

`FacetManager` gains a `setEsbuildService(s)` setter, wired by `NimbusSession` at the same point it constructs `esbuildService` lazily, OR (simpler) `FacetManager` lazily creates its own `EsbuildService(this.vfs)` on first exec.

**Caching:** keep a per-FacetManager `Map<contentHash, cjsCode>` so re-execs don't re-transform.

### Fix C ‚Äî surface the silent-compile error

**File:** `src/facet-manager.ts`
**Functions:** `generateFacetCode`, `generateEntrypointCode` (the inline JS at lines 205‚Äì210 and 383‚Äì388)

Replace:
```js
try { __compiledModules.set(__p, new Function(...)); } catch {}
```
with:
```js
try {
  __compiledModules.set(__p, new Function("exports","require","module","__filename","__dirname", __c));
} catch (__e) {
  __compileFailures.push({ path: __p, error: __e && __e.message ? __e.message : String(__e) });
}
```
where `__compileFailures` is a new module-level array.

Then in `__loadModule` (`src/node-shims.ts:2052+`), when the precompiled function is missing AND we're about to throw the "file was not pre-bundled" error, check `__compileFailures` for a matching entry and include the original SyntaxError message in the thrown error:
```js
const failure = __compileFailures.find(f => f.path === resolvedPath || f.path === normalizedPath);
if (failure) {
  throw new Error("Cannot load module '" + resolvedPath + "': pre-compile failed at facet startup: " + failure.error);
}
throw new Error("Cannot load module '" + resolvedPath + "': file was not pre-bundled. Add it to the VFS bundle.");
```

Side benefit: future regressions (someone adds a new ESM-only package, adds a syntax-incompatible file, etc.) surface the real reason at the user terminal instead of the misleading "not pre-bundled".

## 6. Risk + acceptance criteria

### Risks

| Risk | Likelihood | Mitigation |
|---|---|---|
| ESM transform inflates bundle past `BUNDLE_MAX_ENCODED_BYTES` | medium | Re-eviction loop after transform pass (already in ¬ß2 sketch); fall back to "skip transform if blow-up". |
| `esbuildService.transform` cold-start (~10 ms wasm init + per-call) inflates exec latency | medium | Cache by content hash; lazy-init the service; only transform `.js`/`.mjs` files that pass the regex sniff (typical user app: ~5‚Äì50 files). |
| Regex sniff false-positive on a CJS file containing the substring `export ` in a string literal | low | Sniff strips comments AND only matches `^\s*(import|export)\b` at line start. Strings with embedded `\nexport ` are pathological; if hit, transform-as-CJS yields a CJS rebuild that still works (esbuild emits valid CJS for any input it accepts). |
| `__pathIsFile` change breaks something that depended on the loose `__fileExists` semantics inside `__resolveFile` | low | Other callers of `__fileExists` (`__resolveNodeModule`, `__resolveImportsField`) untouched. `__resolveFile` is the only spot where directory-as-file-key was a bug. |
| `BUNDLE_MAX_ENCODED_BYTES` regression on bundles where many ESM files transform to slightly larger CJS | low | Same re-eviction loop; if it bites, raise the cap (currently in `src/constants.ts`). |
