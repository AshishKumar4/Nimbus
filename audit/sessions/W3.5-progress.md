### Phase C — Build

Three fixes landed in src/:

**Fix A — directory-as-index in `__resolveFile`**
- File: `src/node-shims.ts`
- Added a new strict `__pathIsFile(p)` helper that consults only
  `__vfsBundle` and `__vfsWrites` (no `__vfsDirs` membership, no
  prefix scan). Other callers of `__fileExists` keep the loose
  semantic for "directory exists" probes.
- Reworked `__resolveFile` to use `__pathIsFile` for the empty-extension
  probe so a directory-shaped `base` falls through to `/index.js` etc.
- Unit-validated locally by extracting the three functions out of the
  generated shim string and exercising them against a fixture VFS.
  Tests run during Phase C build (now removed): all 7 cases PASS,
  including the W3-retro `ret/dist/types` directory case.

**Fix B — ESM→CJS pre-pass in `buildPrefetchBundle`**
- File: `src/facet-manager.ts`
- New helpers: `looksLikeEsm(src)` (regex sniff for top-level
  import/export STATEMENTS, comments stripped), `transformEsmInBundle`
  (walks bundle, calls `EsbuildService.transform({format:'cjs'})` on
  matches, in-place mutates bundle), plus a content-hashed transform
  cache `__esmTransformCache` so warm execs don't re-pay wasm dispatch.
- `buildPrefetchBundle` is now `async` and accepts an optional
  `EsbuildService`. `FacetManager.exec` lazy-creates one when needed.
- `NimbusSession.ensureFacetManager` shares its session-level
  esbuildService with the FacetManager (avoids double init).
- The 18-case `looksLikeEsm` regex unit test (run during Phase C
  build, then removed) covered: import statements, named/wildcard/
  default exports, comments, dynamic `import()` (correctly excluded),
  property accesses (correctly excluded). All PASS.

**Fix C — surface silent compile failures**
- Files: `src/facet-manager.ts` (template) + `src/node-shims.ts`
  (`__loadModule`).
- The two duplicated pre-compile loops at facet-template lines 207 +
  385 now record SyntaxErrors into a `__compileFailures` map (instead
  of swallowing them in `try{}catch{}`).
- `__loadModule`'s misleading "file was not pre-bundled" branch now
  consults `__compileFailures` first; if the file failed pre-compile,
  surface the original error: `"Cannot load module 'x.js':
  pre-compile failed at facet startup: <real reason>"`.

Type check: 2 pre-existing baseline errors, identical to W3 retro
(esbuild-wasm/esbuild.wasm declaration; SqliteVFSProvider mount type).
No new errors.

src/ delta: +227 / -6 (3 files). Plan estimate was +92 / -8 — overshot
mostly because the ESM transform cache + helper code grew bigger than
the v1 sketch (extra docstrings + edge-case handling around the regex).
