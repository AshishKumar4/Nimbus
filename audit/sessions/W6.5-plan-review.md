## Must_fix (resolved in v2)

### MF-1. Telemetry "supervisor catches facet rejects" claim is FACTUALLY WRONG
**Location:** v1 §5.3, §5.6, §8.1 C.2.

`src/npm-installer.ts:580–584` discards the original facet error and creates a fresh `new Error("resolver-facet failed: " + msg)`. By the time supervisor code sees the failure, the `__w6_reject` own-property tag is gone and `isRegistryReject(e)` returns false. Default-prod path (facet) blinded.

**Resolution in v2:** §5.3 redesigned. Facet writes events into a `pendingEvents[]` array, returns them on success via a new `ResolveFacetResult.registryEvents` field. Supervisor reads + flushes to sink. The throw-path `reject` event remains a documented incomplete-coverage gap (better than v1's full blindness).

### MF-2. Module-level `_sink` isolate-safety undocumented and untested
**Location:** v1 §5.3.

**Resolution in v2:** §5.5 documents the per-isolate-singleton invariant. New `functional/sink-isolation.mjs` probe spawns concurrent child bun processes with different sinks; asserts no cross-contamination.

### MF-3. §6 transitive-swap-end-to-end probe path doesn't exist
**Location:** v1 §6 + §7.2.

`resolveTree(specs, cache, …)` requires `NpmCache` (SQLite-backed) and `NimbusFacetPool` (DO-isolate scaffolding); no in-memory drop-in.

**Resolution in v2:** §6 split into trio: (6.1) decision-rule unit test on synthetic tree; (6.2) extends W6's symmetric extraction probe with transitive row; (6.3) optional prod e2e via `NIMBUS_W6_5_E2E_PROD=1`.

### MF-4. C.5 "optional" REJECT entries lacked RED probes — TDD violation
**Location:** v1 §8.1 C.5.

**Resolution in v2:** §7.1 adds explicit per-entry probes `sharp-wasm32-rejected.mjs` and `napi-rs-canvas-wasi-rejected.mjs`.

### MF-5. §4 "cosmetic-string-only" claim wrong; _SUMMARY.json unsafe oracle
**Location:** v1 §4 + §7.1 reject-message-quality.mjs.

W6 retro S5: `_SUMMARY.json` records install success not load success. sql.js, @swc/wasm-web, wasm-vips all show `ok:true` while `.out.txt` shows runtime fails.

**Resolution in v2:** §4.1 rewrote oracle to read `<name>.out.txt` directly and check known-fail markers (`ENOENT`, `not pre-bundled`, `MODULE_NOT_FOUND`, `Cannot find module`). §4.2 adds `_SUMMARY.json` honesty pass as a Track 1 prerequisite.
